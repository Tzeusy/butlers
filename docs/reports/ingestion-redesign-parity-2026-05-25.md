## Prototype vs Live Divergences

Comparing the prototype in `pr/overview/ingestion-redesign/` to the live
implementation:

### Deliberate Deviations

1. **OAuth scope list shows "unavailable" state** (not hidden).
   The `connector-oauth-scope-surface` backend capability is not yet implemented.
   The `ScopeList` component renders the explicit "unavailable" state per spec
   AC3 ("unsupported or unavailable state must be rendered explicitly, not hidden").
   The prototype (`ingestion-connector-detail.jsx`) shows a populated scope list
   — this is aspirational and tracked in the oauth-scope-surface OpenSpec change.

2. **Live/Idle status badge** on the Timeline page derives its state from a
   simple 300ms/30s timer rather than an SSE subscription. The prototype shows
   "live" continuously. The spec allows this as "a future iteration can
   subscribe to the SSE stream for precise timing".

3. **Dormant connectors section** renders when `connector-available` endpoint
   returns available-but-unconnected connectors. The fixture mock returns no
   dormant connectors, so the section is empty in test runs. The prototype shows
   example dormant rows.

### No Divergences Found

- Route hierarchy matches spec exactly (4 routes, including `/ingestion/history` redirect).
- Sub-nav (Timeline / Connectors / Filters) uses NavLink, no TabsTrigger.
- Card chrome is absent from all primary ingestion surfaces.
- Dispatch visual language (hairline borders, mono eyebrows, serif voice text) is present.
- Legacy `?tab=` redirects strip the `tab` param and forward to correct sub-routes.
