# Heart and Soul

Why Butlers exists, what it believes, and what it refuses to be.

These documents are the project's doctrine. They govern design decisions,
scope arguments, and feature debates. When in doubt, start here.

## How to Use These

- **Adding a feature?** Check `vision.md` for non-negotiable rules and `v1.md` for scope.
- **Making a design decision?** Check `architecture.md` for structural philosophy.
- **Worried about security?** Check `security.md` for the trust model.
- **Starting development?** Check `development.md` for workflow principles.
- **Arguing about scope?** `v1.md` is the boundary. If it is not listed, it waits.
- **Touching the dashboard?** Read `design-language.md` before adding chrome, tokens, or page archetypes.

## What These Are Not

These are not implementation guides, API references, or deployment runbooks.
For those, see the main [documentation index](../index.md).
